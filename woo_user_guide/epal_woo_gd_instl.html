Installing the ePAL Plugin

This topic exlpains how you can use the guided installation to install the plugin.

Prerequisites

You have deployed the plugin as described in the Uploading the ePAL Plugin topic.

You can create a new user in step 2 of the guided activity or you can sign up for one at the ePAL Dashboard before you start the installation.

Guided Installation Overview

The plugin's guided installation consists of 7 steps. These are displayed in the header during installation. As you complete each step, the circle around each step changes to green. Blue represents the current step.

The steps are as follows:
  1. Enable Taxes and Calculations: Enable taxes and configure some other WooCommerce settings.
  2. API Credentials: Configure your access credentials.
  3. ePAL Tax Configuration: Configure the ePAL tax settings.
  4. Shipping Address: Configuring your shipping address. You can use your store's address if you want to.
  5. Shipping Methods: Configure your shipping methods.
  6. Shippo OAuth Integration(Optional): Confgure you OAuth settings for integrations with Shippo. This step is optional.
  7. Product Configuration: You need to generate and review HS6 codes for your products.

To access this guide at anytime, you can amend your WordPress Admin URL with /wp-admin/admin.php?page=wc-settings&tab=epal&section=setup to open it.

These settings can also be accessed in the ePAL portal.

To finish the setup, complete the following steps:

1. Enable Taxes and Calculations

The first step prompts you to configure the required settings in WooCommerce:

To open the relevant settings page, you can click Open WooCommerce General Settings. The following UI is displayed:

Ensure that the Enable tax rates and calculations checkbox is selected.

2. API Credentials

If you need to, you can create a new ePAL user or you can sign in using an existing one in Step 2:

You can proceed in one of the following ways:
  • If you have an existing user account from the ePAL portal but have not yet retrieved your API credentials, you can enter the email and passwoord and then click Login and Get Credentials.
  • If you want to sign up for an ePAL account now, complete the Email, Password, Full Name, and Company Name fields. Then click SignUp.
  • If you have an existing users and have already retrieved your credentials, you can enter them in the Client IDand Client Secret fields.

Once you have successfully signed in, you may need to reload the page when prompted. After this, the following UI is displayed:

IMAGE

You can click the Test Connecton button to ensure the connection is working. You can also test your connection to Shippo, if you has been configured.

To logout, click Logout.

3. Configure Tax Settings

Now, you can configure your WooCommerce tax settings.

To display the following UI, click Open Woo Commerce Tax Settings:

Complete the following fields:
Table 1. ePAL Tax Calculation UI Fields
Field Description
Prices entered with tax

If your store uses prices that include tax, select the Yes, I will enter prices inclusive of tax radio buttton.

If your store uses prices that exclude tax, select the No, I will enter prices exclusive of tax radio buttton.
Calculate tax based on
To configure the information that is used for tax calculation, choose one of the following options:
  • Customer shipping address: Use the address that the customer provides for receiving the goods.
  • Customer billing address:Use the billing addresss that the customer specified.
  • Shop base address: Use the store's default address.
Shipping tax class To configure the tax classification that will be used to calculate the tax on shipping, choose one of the following options:
  • Shipping tax class based on the cart: Choose this option if you do not want to set the behavior and use the default option instead. Tax is calculated based ong the goods in the transaction.
  • Standard: The standard tax rate is applied.
  • Reduced rate: The reduced tax rate is applied.
  • Zero rate: A tax rate of zero is applied.
Rouding

If you select the Round tax at subtotal level, instead of rounding per line button, rounding will be done for each subtotal rather than each line.

If you do not select the button, rounding will continue to be done on each line.
Additional Tax Classes You can list any tax classes that you might need but could not specify in the Shipping tax class setting. You may want to add ones for Reduced and Zero Rated tax classes.
Display prices in the shop
To specify whether you want to display tax inclusive or exclusive prices in your store, choose one of the following options:
  • Including tax: The displayed prices include taxes.
  • Excluding tax: The displayed prices exclude taxes.
Display prices during cart and checkout
To specify whether you want to display tax inclusive or exclusive prices in your cart and during checkout, choose one of the following options:
  • Including tax: The displayed prices include taxes.
  • Excluding tax: The displayed prices exclude taxes.
Price display suffix You can use this to specify a free text suffix for prices. For example, you might specify (incl. VAT) to specify VAT inclusive prices.
Display tax totals
Choose how you want tax totals to display. You can choose from one of the following:
  • As a single total: Tax is displayed in the total>
  • Itemized: Tax is displayed per item.

Next, you need to configure the ePAL tax settings that are displayed in the third step:

Complete the following fields:
